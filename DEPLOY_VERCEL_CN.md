# Vercel 部署说明

## 当前项目已完成

- 已配置 `vercel.json`
- 已指定 `Vite` 构建命令和输出目录
- 已添加 `.vercelignore`
- 已将页面图片改为本地海报优先，减少中国大陆访问时对外部图片源的依赖

## 最快上线方式

1. 注册并登录 Vercel
2. 把当前项目上传到 GitHub
3. 在 Vercel 中选择 `Add New -> Project`
4. 导入这个 GitHub 仓库
5. 保持默认构建设置，直接点击部署

## 命令行方式

在项目根目录执行：

```bash
npx vercel
```

首次会要求登录并把本地目录绑定到一个 Vercel 项目。

正式发布可执行：

```bash
npx vercel --prod
```

## 中国大陆访问建议

- 不要直接长期使用 `*.vercel.app` 域名，尽量绑定你自己的域名
- 优先自托管关键资源，避免依赖外部字体、脚本、图片
- 如果需要更稳定的中国大陆访问，建议额外准备一个面向国内线路的镜像站点

## 重要说明

Vercel 可以部署这个项目，但不能单独保证中国大陆网络下始终稳定可访问。  
如果需要更高把握，至少应同时做到：

- 自定义域名
- 本地静态资源优先
- 尽量减少第三方外链
- 必要时增加国内镜像或国内合规托管
